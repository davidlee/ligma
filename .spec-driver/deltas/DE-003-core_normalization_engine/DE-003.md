## 2. Motivation
The normalization layer is the core product value. This delta implements the foundational normalization pipeline: type classification, node normalization, layout extraction, appearance normalization, and text normalization. These are the highest-value transformations that make raw Figma JSON implementation-ready.

## 3. Scope & Objectives
- **Primary Outcomes**:
  - Zod schemas for NormalizedNode and sub-types (FR-005)
  - Type classifier mapping Figma types → NormalizedNodeType (FR-006)
  - Node normalizer producing recursive NormalizedNode trees (FR-005)
  - Layout normalizer: mode/sizing/alignment/padding/gap/wrap/constraints (FR-008)
  - Appearance normalizer: fills/strokes/effects/corner radius (FR-005)
  - Text normalizer: content/style/color/semantic kind/truncation (FR-009)
  - Representation efficiency: fixed field set + ≤2.0x size ceiling (NF-001, RE-002)
  - All heuristics as isolated pure functions with tests (NF-003)
- **Dependencies**: DE-002 (project scaffold, raw types, Figma client)

## 4. Out of Scope
- Role inference heuristics (DE-004)
- Component/instance metadata (DE-004)
- Variable/token extraction (DE-004)
- Asset classification (DE-004)
- Outline/context generation (DE-005)

## 5. Approach Overview
- **System Touchpoints**: `src/normalize/*`, `src/schemas/normalized.ts`, `src/orchestrate.ts`
- **Key Changes**:
  - `schemas/normalized.ts` — Zod schemas for all normalized types (full skeleton, DE-004 fields null)
  - `normalize/classify.ts` — type classification (incl. image/mask detection)
  - `normalize/bounds.ts` — bounds extraction from absoluteBoundingBox
  - `normalize/node.ts` — core recursive normalizer with compositional diagnostics
  - `normalize/layout.ts` — layout extraction (horizontal, vertical, grid, absolute)
  - `normalize/appearance.ts` — appearance extraction (fills, strokes, effects, corner radius)
  - `normalize/text.ts` — text content and style extraction (DimensionValue for unitful fields)
  - `normalize/index.ts` — entry point, root context creation
  - `orchestrate.ts` — adds normalizedNode to OrchestrateResult (DEC-008)

## 6. Verification Strategy
- Unit tests per normalize module (classify, bounds, layout, appearance, text, node)
- Fixture tests with raw Figma JSON → expected normalized output
- Unsupported/unknown input tests (EMOJI/PATTERN paints, TEXTURE/NOISE effects → warnings)
- Representation efficiency tests (NF-001, RE-002)
- Golden snapshot tests for representative UI patterns

## 7. Risks & Mitigations
- **Risk**: Figma auto-layout property mapping is ambiguous – *Likelihood*: medium – *Impact*: medium – *Mitigation*: Conservative mapping with diagnostics, test against real frames
- **Risk**: Text style extraction misses edge cases – *Likelihood*: medium – *Impact*: low – *Mitigation*: Fixture tests for mixed-style text
