# DR-003 – Core normalization engine

## 1. Executive Summary
- **Delta**: [DE-003](./DE-003.md)
- **Status**: draft
- **Last Updated**: 2026-03-10
- **Synopsis**: Implement the foundational normalization pipeline that transforms raw Figma JSON into a compact, implementation-oriented `NormalizedNode` tree. Covers type classification, bounds, layout, appearance, and text extraction — the structural core that all downstream deltas build on.

## 2. Problem & Constraints
- **Current Behaviour**: `orchestrate()` returns raw Figma JSON as `unknown`. No structural transformation. Agents must parse noisy, verbose raw API output.
- **Drivers / Inputs**: SPEC-001 FR-005/FR-006, PROD-001 FR-005/FR-006/FR-008/FR-009, NF-001, NF-003. The normalization layer is the core product value (brief §Purpose).
- **Constraints / Guardrails**: All extractors must be pure functions (NF-003). Output must be >50% smaller than raw input (NF-001). No cross-extractor dependencies in DE-003. Node 18+, Zod for schema validation. ADR-001 zero lint warnings.
- **Out of Scope**: Role inference heuristics (DE-004). Component/instance metadata extraction (DE-004). Variable/token binding extraction (DE-004). Asset classification (DE-004). Used-token summary (DE-004). Outline generation (DE-005). context.md generation (DE-005). writeOutput/manifest changes (DE-005). Mixed-style text (characterStyleOverrides).

## 3. Architecture Intent

### Target Outcomes
- `normalize(rawNode: FigmaNode): NormalizedNode` — pure function, recursive tree transformation
- Every raw Figma node type classified into implementation-relevant `NormalizedNodeType`
- Layout, appearance, and text extracted into compact, typed representations
- Diagnostics track provenance: source type, omitted fields, warnings, confidence
- Normalized output >50% smaller than raw input for typical frames

### Module Data Flow

```
FigmaNode (raw, Zod-validated, passthrough)
  │
  ├─ classify(node) → NormalizedNodeType
  │
  └─ normalizeNode(raw, context) → NormalizedNode
       │
       ├─ extractBounds(raw) → ExtractorResult<Bounds | null>
       ├─ extractLayout(raw) → ExtractorResult<NormalizedLayout | null>
       ├─ extractAppearance(raw) → ExtractorResult<NormalizedAppearance | null>
       ├─ extractText(raw) → ExtractorResult<NormalizedText | null>  (text only)
       ├─ buildHierarchy(context) → Hierarchy
       ├─ aggregate diagnostics from all extractors
       │
       └─ recurse: children.map(child => normalizeNode(child, childContext))
```

### Guiding Principles
- Each extractor is a pure function: raw node in, typed result + diagnostics out
- No cross-extractor dependencies in DE-003 (DE-004 may add a post-processing stage)
- `node.ts` is the composition point — it fans out to extractors and aggregates results
- `index.ts` is trivially thin — creates root context, delegates to `normalizeNode`
- Diagnostics are compositional: each extractor contributes, `node.ts` aggregates
- Absence semantics: `null` = not attempted/not applicable, `"unknown"` = attempted but indeterminate

### Shared Raw Field Accessor (DEC-017)

All extractors access passthrough fields via a shared utility, not independent casts:

```ts
// src/normalize/raw-helpers.ts (or inline in node.ts)
function getRawProperty<T>(node: FigmaNode, key: string, defaultValue: T): T {
  const value = (node as Record<string, unknown>)[key]
  return value === undefined ? defaultValue : value as T
}
```

This isolates the Zod passthrough cast to a single location.

### Null vs Populated Semantics (DEC-018)

- `layout: null` — only for `DOCUMENT` and `CANVAS` node types (cannot have layout properties)
- `layout: { mode: "none", ... }` — for all other nodes, even when no auto-layout is present
- `appearance: null` — only for `DOCUMENT` and `CANVAS` node types
- `appearance: { fills: [], strokes: [], ... }` — for all other nodes, even when no visible fills/strokes
- `text: null` — for all non-text nodes. Always populated for `type === "text"`

**Extractor contract**: `node.ts` gates calls — extractors are never called for document/page nodes. When called, extractors always return a populated default object (not `null`). For example, `extractLayout` returns `{ mode: "none", sizing: { horizontal: "fixed", vertical: "fixed" }, align: { main: "unknown", cross: "unknown" }, padding: null, gap: null, wrap: null, grid: null, constraints: null, position: null, clipsContent: null }` as the floor. Extractors never need to reason about node-type applicability.

### Error Strategy (DEC-019)

`normalize()` is warning-based with graceful degradation. Individual extractor failures produce diagnostic warnings, not exceptions. `NormalizationError` is thrown only on catastrophic failure — e.g., root node missing required `id`/`name`/`type` fields. This matches the pipeline's partial-failure philosophy (FR-017).

### Integration Boundary
- `normalize()` wired into `orchestrate.ts` — adds `normalizedNode: NormalizedNode` to `OrchestrateResult`
- `orchestrate()` calls `normalize(fileResponse.document)` — `fetchNode` already extracts the scoped selected node via `nodes[nodeId].document` from the Figma API response
- No changes to `writeOutput` or manifest — DE-005 handles output pipeline

### NormalizedNodeType

```ts
type NormalizedNodeType =
  | "document" | "page" | "frame" | "group"
  | "component" | "instance" | "variant-set"
  | "text" | "shape" | "vector" | "image" | "line"
  | "boolean-operation" | "mask" | "section" | "unknown"
```

Classification rules:
- `DOCUMENT` → `"document"`, `CANVAS` → `"page"`, `FRAME` → `"frame"`, `GROUP` → `"group"`
- `COMPONENT` → `"component"`, `COMPONENT_SET` → `"variant-set"`, `INSTANCE` → `"instance"`
- `TEXT` → `"text"`, `LINE` → `"line"`, `SECTION` → `"section"`
- `RECTANGLE`/`ELLIPSE`/`POLYGON`/`STAR` → `"shape"`
- `VECTOR` → `"vector"`, `BOOLEAN_OPERATION` → `"boolean-operation"`
- `isMask === true` → `"mask"` (overrides base type)
- `IMAGE` fill present → `"image"` (semantic implementation category, not a raw Figma node kind — Figma has no IMAGE node type; a rectangle with an image fill is still structurally a rectangle, but for implementation purposes it is an image). Classify does presence-only check (`fills.some(f => f.type === "IMAGE")`), not visibility-filtered — appearance extractor handles visibility and detailed paint normalization.
- Unmapped → `"unknown"`

### NormalizedRole

```ts
type NormalizedRole =

  | "list" | "list-item" | "modal" | "navigation"
```

`null` in DE-003. No `"unknown"` in the enum — `null` carries that meaning.

### NormalizedLayout

```ts
type LayoutMode = "none" | "horizontal" | "vertical" | "grid" | "absolute"
type SizingMode = "fixed" | "fill" | "hug" | "unknown"
type MainAlign = "start" | "center" | "end" | "space-between" | "unknown"
type CrossAlign = "start" | "center" | "end" | "stretch" | "baseline" | "unknown"
type ConstraintMode = "min" | "max" | "center" | "stretch" | "scale" | "unknown"

type NormalizedGrid = {
  rows: number | null
  columns: number | null
  rowGap: number | null
  columnGap: number | null
}

type NormalizedLayout = {
  mode: LayoutMode
  sizing: { horizontal: SizingMode; vertical: SizingMode }
  align: { main: MainAlign; cross: CrossAlign }
  padding: { top: number; right: number; bottom: number; left: number } | null
  gap: number | null
  wrap: boolean | null
  grid: NormalizedGrid | null
  constraints: { horizontal: ConstraintMode; vertical: ConstraintMode } | null
  position: { x: number; y: number; positioning: "flow" | "absolute" } | null
  clipsContent: boolean | null
}
```

Layout mapping:
- `layoutMode: "HORIZONTAL"` → `mode: "horizontal"`, `"VERTICAL"` → `"vertical"`, `"GRID"` → `"grid"`, absent/`"NONE"` → `"none"` or `"absolute"`
- `"absolute"` mode: only `FRAME`, `COMPONENT`, or `INSTANCE` with no auto-layout (`layoutMode` absent or `"NONE"`) **and** at least one child. `GROUP` always gets `"none"` (groups are structural, not positional containers). `SECTION` gets `"none"`. Nodes with zero children get `"none"`.
- Grid: populate `grid` subobject from `gridRowCount`, `gridColumnCount`, `gridRowGap`, `gridColumnGap`. Per-child grid placement properties (e.g., column/row span) are a known gap — acceptable for DE-003, may be added in a follow-up. Note: for `mode: "grid"`, `align.main`/`align.cross` still map from `primaryAxisAlignItems`/`counterAxisAlignItems` (same API fields), though the axis semantics differ from flexbox.
- Sizing: prefer `layoutSizingHorizontal`/`layoutSizingVertical` when present; fall back to `primaryAxisSizingMode`/`counterAxisSizingMode` (axis-relative)
- `"fill"` sizing outside auto-layout parent → map to `"fixed"` with diagnostic warning
- Padding: `paddingTop`/`Right`/`Bottom`/`Left`; null if all zero and no auto-layout
- Alignment: `primaryAxisAlignItems` → `align.main`: `"MIN"` → `"start"`, `"CENTER"` → `"center"`, `"MAX"` → `"end"`, `"SPACE_BETWEEN"` → `"space-between"`. `counterAxisAlignItems` → `align.cross`: `"MIN"` → `"start"`, `"CENTER"` → `"center"`, `"MAX"` → `"end"`, `"BASELINE"` → `"baseline"`. Unmapped values → `"unknown"`.
- Wrap: `layoutWrap: "WRAP"` → `true`, `"NO_WRAP"` → `false`, absent → `null`. `counterAxisSpacing` is out of DE-003 scope — add to `omittedFields` when `wrap` is `true`.
- Position: `x`/`y` from raw node properties (always present on positioned children). `layoutPositioning: "ABSOLUTE"` → `positioning: "absolute"`, `"AUTO"` or absent → `positioning: "flow"`. Do NOT use `relativeTransform` — requires `geometry=paths`. Diagnostic warning if `x`/`y` absent.
- Constraints: `"MIN"` → `"min"`, `"MAX"` → `"max"`, `"CENTER"` → `"center"`, `"STRETCH"` → `"stretch"`, `"SCALE"` → `"scale"`. Unmapped values → `"unknown"` with diagnostic warning.

### NormalizedAppearance

```ts
type NormalizedPaint = {
  kind: "solid" | "gradient" | "image" | "video" | "unknown"
  visible: boolean
  color: string | null
  opacity: number | null          // paint-level opacity (per-fill/stroke)
  gradientStops: GradientStop[] | null
  tokenRef: string | null         // populated in DE-004
  imageRef: string | null
}

type StrokeAlign = "inside" | "outside" | "center"

type NormalizedStroke = NormalizedPaint & {
  weight: number | null
  align: StrokeAlign | null
}

type CornerRadius =
  | { uniform: true; radius: number }
  | { uniform: false; topLeft: number; topRight: number;
      bottomRight: number; bottomLeft: number }

type EffectKind = "drop-shadow" | "inner-shadow" | "layer-blur"
  | "background-blur" | "unknown"

type NormalizedEffect = {
  kind: EffectKind
  visible: boolean
  color: string | null
  offset: { x: number; y: number } | null
  radius: number | null
  spread: number | null
}

type NormalizedAppearance = {
  fills: NormalizedPaint[]
  strokes: NormalizedStroke[]
  cornerRadius: CornerRadius | null
  effects: NormalizedEffect[]
  blendMode: string | null
  opacity: number | null  // node/container-level opacity; distinct from per-paint opacity
}
```

Paint mapping:
- `"SOLID"` → `"solid"`, `"GRADIENT_LINEAR"`/`"GRADIENT_RADIAL"`/`"GRADIENT_ANGULAR"`/`"GRADIENT_DIAMOND"` → `"gradient"`, `"IMAGE"` → `"image"`, `"VIDEO"` → `"video"`
- `"EMOJI"`, `"PATTERN"` → `"unknown"` with diagnostic warning
- Color: `{ r: 0-1, g: 0-1, b: 0-1, a?: 0-1 }` → `#RRGGBB` or `#RRGGBBAA` when `a < 1`. Conversion: `Math.round(r * 255)` for deterministic output (DEC-020).
- Gradient stops: `[{ position, color: hex }]`
- Corner radius: uniform if single `cornerRadius` value; per-corner from `rectangleCornerRadii`
- Effects: `"DROP_SHADOW"` → `"drop-shadow"`, `"INNER_SHADOW"` → `"inner-shadow"`, `"LAYER_BLUR"` → `"layer-blur"`, `"BACKGROUND_BLUR"` → `"background-blur"`, `"TEXTURE"`/`"NOISE"` → `"unknown"` with diagnostic warning
- `blendMode`: pass through lowercase; `null` only if `"PASS_THROUGH"` (the group/frame default). `"NORMAL"` is preserved as `"normal"` — a group with explicit `NORMAL` instead of `PASS_THROUGH` changes blend isolation semantics.
- `opacity`: node-level; distinct from per-paint opacity on each fill/stroke
- `strokeWeight`: per-node property applied to all strokes. Each `NormalizedStroke` entry carries the same `weight` value.

### NormalizedText

```ts
type TextSemanticKind = "heading" | "label" | "body" | "caption" | "button" | "unknown"

type NormalizedText = {
  content: string
  charactersLength: number
  style: {
    fontFamily: string | null
    fontWeight: number | null
    fontSize: number | null
    lineHeight: DimensionValue | null
    letterSpacing: DimensionValue | null
    textCase: string | null
    textAlignHorizontal: string | null
    textAlignVertical: string | null
```

Text mapping:
- `characters` → `content`, `characters.length` → `charactersLength`
- `style.lineHeightUnit`: `"PIXELS"` → `{ unit: "px", value: lineHeightPx }`, `"FONT_SIZE_%"` → `{ unit: "percent", value: lineHeightPercentFontSize }`, `"INTRINSIC_%"` → `{ unit: "auto", value: null }`
- `style.letterSpacing`: always numeric px (no unit field exists in API). Normalized as `{ unit: "px", value: <number> }`
- `textCase`: `"UPPER"` / `"LOWER"` / `"TITLE"` / `"ORIGINAL"` / `"SMALL_CAPS"` / `"SMALL_CAPS_FORCED"` → lowercase
- Text alignment: `"LEFT"` / `"CENTER"` / `"RIGHT"` / `"JUSTIFIED"` → lowercase
- `color`: first visible fill's color → hex; `null` if no visible fills
- `textTruncation: "ENDING"` → `truncation.ellipsis: true`; `maxLines` direct
- `semanticKind`: `"unknown"` in DE-003 (attempted, indeterminate). DE-004 infers.
- `tokenRefs`: `[]` in DE-003. DE-004 populates.
- Mixed-style text (`characterStyleOverrides` + `styleOverrideTable`): not in DE-003 scope. Default `style` only.

### `node.ts` composition

```ts
const SKIP_EXTRACTORS = new Set(["document", "page"])  // DEC-018: DOCUMENT/CANVAS cannot have layout/appearance
const EMPTY_RESULT: ExtractorResult<null> = { value: null, warnings: [], omittedFields: [] }

function normalizeNode(raw: FigmaNode, context: NormalizeContext): NormalizedNode {
  const type = classify(raw)
  const isText = type === "text"
  const skipExtractors = SKIP_EXTRACTORS.has(type)

  const boundsResult = skipExtractors ? EMPTY_RESULT : extractBounds(raw)
  const layoutResult = skipExtractors ? EMPTY_RESULT : extractLayout(raw)
  const appearanceResult = skipExtractors ? EMPTY_RESULT : extractAppearance(raw)
  const textResult = isText ? extractText(raw) : null

  const allWarnings = [
    ...boundsResult.warnings,
    ...layoutResult.warnings,
    ...appearanceResult.warnings,
    ...(textResult?.warnings ?? []),
  ]
  const allOmitted = [
    ...boundsResult.omittedFields,
    ...layoutResult.omittedFields,
    ...appearanceResult.omittedFields,
    ...(textResult?.omittedFields ?? []),
  ]

  const childContext: NormalizeContext = {
    parentId: raw.id,
    depth: context.depth + 1,
    path: [...context.path, { id: raw.id, name: raw.name, type: raw.type }],
  }

  return {
    id: raw.id,
    name: raw.name,
    type,
    role: null,
    visible: getRawProperty(raw, 'visible', true) !== false,
    bounds: boundsResult.value,
    rotation: extractRotation(raw),
    hierarchy: {
      parentId: context.parentId,
      depth: context.depth,
      childCount: raw.children?.length ?? 0,
      path: context.path,
    },
    layout: layoutResult.value,
    appearance: appearanceResult.value,
    text: textResult?.value ?? null,
    component: null,
    variables: null,
    asset: null,
    semantics: DEFAULT_SEMANTICS,
    children: (raw.children ?? []).map(c => normalizeNode(c, childContext)),
    diagnostics: {
      sourceNodeType: raw.type,
      omittedFields: allOmitted,
      warnings: allWarnings,
      confidence: allWarnings.length > 0 ? "medium" : "high",  // "low" reserved for DE-004 heuristic uncertainty
    },
  }
}
```

`extractRotation` is a private helper in `node.ts`:
- `extractRotation(raw)`: uses `getRawProperty(raw, 'rotation', null)`. Returns `null` when rotation is 0 or absent (means "absent or default", not "unknown").

## 8. Design Decisions & Trade-offs

- **DEC-007**: Full `NormalizedNode` skeleton defined in DE-003; DE-004 fields present but null. Prevents type-breaking changes downstream. Near-zero cost vs defining only DE-003 fields.
- **DEC-008**: `normalize()` wired into `orchestrate()`, adds `normalizedNode` to `OrchestrateResult`. `writeOutput`/manifest unchanged (DE-005 scope). Proves integration without expanding blast radius.
- **DEC-009**: Diagnostics compositional — each extractor returns `{ value, warnings, omittedFields }`. `node.ts` aggregates. Confidence `"high"` when clean, `"medium"` on any extractor warning. Avoids re-deriving omission context in a late pass.
- **DEC-010**: `classify()` takes full `FigmaNode`, not just type string. Enables image detection (shapes with image fills) and mask detection (`isMask`). Near-zero cost vs string-only.
- **DEC-011**: `lineHeight` and `letterSpacing` use `DimensionValue` (`{ unit, value }`), not `string | number`. Internal normalization choice — Figma semantics are unitful and `"auto"` is materially different from pixels.
- **DEC-012**: Grid layout first-class. `LayoutMode` includes `"grid"`, `NormalizedLayout` has `grid: NormalizedGrid | null` subobject. Figma's grid auto-layout has distinct fields (`gridRowCount`, `gridColumnCount`, `gridRowGap`, `gridColumnGap`) that collapse badly into generic horizontal/vertical.
- **DEC-013**: Child position from raw `x`/`y` properties, not `relativeTransform` (requires `geometry=paths` which DE-002 doesn't request by default). Best-effort with diagnostic warning if absent.
- **DEC-014**: Unsupported paint types (`EMOJI`, `PATTERN`) and effect types (`TEXTURE`, `NOISE`) map to `"unknown"` with diagnostic warnings, not errors. Graceful degradation.
- **DEC-015**: Text extraction limited to default `style` in DE-003. Mixed-style (`characterStyleOverrides` + `styleOverrideTable`) deferred — separate concern, significant complexity.
- **DEC-016**: Absence semantics contract. `null` = not attempted or not applicable. Enum `"unknown"` = attempted but indeterminate. `role: null` (DE-003 doesn't attempt inference). `type: "unknown"` (valid classification for unmapped types). `text.semanticKind: "unknown"` (DE-003 attempts text extraction, can't determine kind).

- **DEC-017**: Shared `getRawProperty<T>(node, key, defaultValue)` utility isolates Zod passthrough field access. All extractors use this instead of independent casts. Single location for the unsafe type narrowing.
- **DEC-018**: `layout` and `appearance` are `null` only for node types that cannot have them (`DOCUMENT`, `CANVAS`). For all other nodes, a populated object with defaults is always present. Simplifies consumer code — no need to check both null and empty. `node.ts` enforces this gate (not the extractors) — extractors are not aware of node types.
- **DEC-019**: `normalize()` is warning-based with graceful degradation. `NormalizationError` thrown only on catastrophic failure (malformed root node). Matches pipeline partial-failure philosophy (FR-017).
- **DEC-020**: Color conversion uses `Math.round(r * 255)` for deterministic hex. Rotation in degrees; `null` = absent or zero default, not "unknown". `"low"` confidence reserved for DE-004 heuristic uncertainty. `blendMode`: `null` only for `PASS_THROUGH` (group/frame default); `NORMAL` preserved as `"normal"` (semantically distinct).
