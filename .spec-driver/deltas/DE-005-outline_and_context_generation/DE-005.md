## 1. Summary & Context
- **Product Spec**: PROD-001 – Figma fetch and normalization pipeline
- **Technical Spec**: SPEC-001 – outline-generation and output-pipeline capabilities
- **Implementation Plan**: [IP-005](./IP-005.md)
- **Change Drivers**: DE-001 planning delta

## 2. Motivation
Agents need two things beyond normalized JSON: a sparse outline for quick navigation (deciding what to expand) and a concise `context.md` as the primary entrypoint. This delta produces both, completing the agent-facing artifact bundle.

## 3. Scope & Objectives
- **Primary Outcomes**:
  - JSON outline generator from normalized tree (FR-011)
  - XML outline generator with compact element-per-node format (FR-011)
  - `context.md` generator with source, visual, structural summary, children, tokens, assets, notes (FR-012)
  - Deterministic output — identical inputs → identical outputs (NF-002)
- **Dependencies**: DE-003 (normalized tree structure). Benefits from DE-004 (richer role/token data in context.md) but can proceed without it.

## 4. Out of Scope
- Normalization (DE-003, DE-004)
- Selective expansion (DE-006)
- Caching (DE-006)

## 5. Approach Overview
- **Design Revision**: [DR-005](./DR-005.md)
- **Design Boundary**: Projection/output only — no NormalizedNode schema or extractor changes
- **New Files**:
  - `src/schemas/outline.ts` — OutlineNode Zod schema + interface
  - `src/normalize/outline.ts` — `buildOutline()` + `outlineToXml()`
  - `src/output/context-md.ts` — `generateContextMd()` agent briefing
- **Modified Files**:
  - `src/schemas/manifest.ts` — 5 output fields become required (DEC-024)
  - `src/output/write.ts` — writes normalized JSON, outlines, context.md
  - `src/orchestrate.ts` — produces outline + context.md, threads `includeHidden`
  - `src/config.ts` — adds `includeHidden` option
  - `src/cli.ts` — adds `--include-hidden` flag (DEC-025)
- **Key Design Decisions**: DEC-021 (hidden node filtering), DEC-022 (important children tiering), DEC-023 (implementation note generators), DEC-024 (manifest required fields), DEC-025 (`--include-hidden` end-to-end)

## 6. Verification Strategy
- VT-018: Outline projection (filtering, childCount, includeHidden)
- VT-019: XML serialization (element naming, escaping, stable attribute order)
- VT-020: context.md (section generators, tiering, golden snapshot)
- VT-021: Determinism (byte-identical output)
- VT-006, VT-012: Regression (manifest schema, orchestrate result)

## 7. Risks & Mitigations
- **Risk**: context.md implementation notes overclaim certainty – *Likelihood*: medium – *Impact*: medium – *Mitigation*: Directive language tied to evidence; conservative language tests (VT-020)
- **Risk**: Important children tiering produces surprising order – *Likelihood*: low – *Impact*: low – *Mitigation*: Document order as stable tie-breaker; no secondary heuristics
