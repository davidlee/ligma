## 1. Summary & Context
- **Product Spec**: PROD-001 – Figma fetch and normalization pipeline
- **Technical Spec**: SPEC-001 – normalization-engine capability (role inference, components, variables, assets)
- **Implementation Plan**: [IP-004](./IP-004.md)
- **Change Drivers**: DE-001 planning delta

## 2. Motivation
Builds the high-value semantic layer on top of DE-003's core normalization. Role inference helps agents understand what UI elements are (button, input, heading). Variable/token extraction distinguishes literal values from design-token-bound values. Component metadata tells agents what's reusable. Asset classification advises what should be exported vs coded.

## 3. Scope & Objectives
- **Primary Outcomes**:
  - Role inference heuristics: container, stack, button, input, icon, heading, etc. (FR-007)
  - Component/instance metadata extraction: componentId, variants, property references (FR-005 component part)
  - Variable/token binding extraction: boundVariables → NormalizedVariableBindings (FR-010)
  - Asset classification: image fills, vector complexity, naming patterns (FR-005 asset part)
  - Diagnostics: confidence scores, warnings, provenance for all inferences
- **Dependencies**: DE-003 (core normalization, NormalizedNode structure)

## 4. Out of Scope
- Core type classification, layout, text normalization (DE-003)
- Outline/context generation (DE-005)
- Selective expansion (DE-006)

## 5. Approach Overview
- **System Touchpoints**: `src/normalize/heuristics.ts`, `src/normalize/components.ts`, `src/normalize/variables.ts`, `src/normalize/assets.ts`
- **Key Changes**:
  - `heuristics.ts` — role inference rules as pure functions
  - `components.ts` — component/instance info extraction
  - `variables.ts` — boundVariables + explicitVariableModes mapping
  - `assets.ts` — asset classification with export suggestions
  - Wire into `normalize/index.ts` pipeline

## 6. Verification Strategy
- Unit tests for each heuristic rule (button detection, input detection, icon detection, etc.)
- Unit tests for component metadata extraction
- Unit tests for variable binding extraction
- Fixture tests with component instances, token-bound nodes, icon groups
- Confidence score validation

## 7. Risks & Mitigations
- **Risk**: Heuristics produce false positives – *Likelihood*: medium – *Impact*: low – *Mitigation*: Conservative defaults (`unknown`), confidence scoring, fixture tests
- **Risk**: Variable resolution requires additional API calls not yet available – *Likelihood*: low – *Impact*: medium – *Mitigation*: Surface raw variable IDs when resolution unavailable
