## 2. Motivation
Builds the high-value semantic layer on top of DE-003's core normalization. Role inference helps agents understand what UI elements are (button, input, heading). Variable/token extraction distinguishes literal values from design-token-bound values. Component metadata tells agents what's reusable. Asset classification advises what should be exported vs coded.

## 3. Scope & Objectives
- **Primary Outcomes**:
  - Role inference heuristics: 13-rule priority chain covering image, icon, button, input, heading, label, body-text, divider, grid, stack, card, container, and fallback (FR-007)
  - Text semantic kind inference: heading/label/body/caption/button/unknown, derived from role with independent fallback (FR-009)
  - Component/instance metadata extraction: kind, componentId, propertyValues (instances), propertyReferences, isReusable (FR-005)
  - Variable/token per-node binding extraction: boundVariables Ôćĺ NormalizedVariableBindings with field path, tokenId, resolvedType (FR-010)
  - Used-token summary aggregation: post-normalization tree walk Ôćĺ tokens-used.json (FR-016)
  - Asset classification: image fills (high confidence), vector complexity (medium, heuristic), naming reinforcement (FR-005)
  - Semantic flag derivation: 6 boolean flags derived from role/type/component (FR-007)
  - Diagnostics: confidence bands (high/medium/low), provenance warnings for all inferences
- **Prep work** (non-breaking refactors):
  - Unified `AnalysisResult<T>` / `ExtractorResult<T>` type hierarchy with explicit confidence
  - `colorToHex()` dedup Ôćĺ `raw-helpers.ts`
  - Schema rename: `variantProperties` Ôćĺ `propertyValues`
- **Dependencies**: DE-003 (core normalization, NormalizedNode structure, extractor pattern)

## 5. Approach Overview

Three-layer architecture per DR-004:

1. **Extraction** (raw Ôćĺ typed fragment): New extractors `extractComponent`, `extractVariables`, `extractAsset` follow DE-003's pure-function extractor pattern. Existing extractors gain explicit confidence.
2. **Assembly**: `node.ts` wires all extractors, recurses children, aggregates diagnostics.
3. **Inference** (assembled node Ôćĺ semantic fields): Top-down post-pass via `applyInferencesRecursive`. New `infer/` subdirectory with `inferRole`, `inferTextKind`, `inferSemantics`, shared `signals` helpers. Operates on narrow `InferenceInput` type.
4. **Summary** (post-normalization): `aggregateTokensUsed` in `src/summary/`, called from `orchestrate.ts`.

- **System Touchpoints**:
  - `src/normalize/components.ts` ÔÇö component/instance metadata extraction
  - `src/normalize/variables.ts` ÔÇö per-node variable binding extraction
  - `src/normalize/assets.ts` ÔÇö asset classification
  - `src/normalize/infer/` ÔÇö role, text-kind, semantics inference + signal helpers
  - `src/normalize/node.ts` ÔÇö pipeline wiring, extractionÔćĺassemblyÔćĺinference
  - `src/normalize/raw-helpers.ts` ÔÇö `AnalysisResult<T>`, `colorToHex()` shared utility
  - `src/summary/tokens-used.ts` ÔÇö post-normalization token aggregation
  - `src/schemas/tokens-used.ts` ÔÇö summary artifact schemas
  - `src/schemas/normalized.ts` ÔÇö `propertyValues` rename
  - `src/orchestrate.ts` ÔÇö tokens-used.json artifact output

## 7. Risks & Mitigations
- **Risk**: Heuristics produce false positives ÔÇô *Likelihood*: medium ÔÇô *Impact*: low ÔÇô *Mitigation*: Conservative defaults (null role, not overconfident unknown), confidence scoring, priority ordering, fixture tests
- **Risk**: Variable resolution requires additional API calls not yet available ÔÇô *Likelihood*: low ÔÇô *Impact*: medium ÔÇô *Mitigation*: Surface raw variable IDs; tokenName/collectionId accept null; explicitModes node-level only
- **Risk**: NF-001 2.0x ceiling breached after populating null fields ÔÇô *Likelihood*: low ÔÇô *Impact*: medium ÔÇô *Mitigation*: Monitor per-field size contribution; investigate before relaxing constraint
- **Risk**: `AnalysisResult<T>` refactor introduces regressions in DE-003 extractors ÔÇô *Likelihood*: low ÔÇô *Impact*: medium ÔÇô *Mitigation*: Atomic change with all existing tests updated simultaneously
