## 1. Summary
- **Delta**: DE-009 — Asset export pipeline
- **Specs Impacted**: SPEC-001
- **Problems / Issues**: IMPR-005
- **Desired Outcome**: `artifacts/assets/` contains exported files for detected assets; manifest tracks them; per-asset failure is non-blocking.

## 2. Context & Constraints
- **Current Behaviour**: Asset detection works (FR-011), but `orchestrate.ts` hardcodes `assets: []`. No collection, fetch, or write logic.
- **Target Behaviour**: Collect export-suggested nodes → batch-fetch via Figma image export → write to `assets/` → populate manifest.
- **Dependencies**: None — builds on existing detection and cache infrastructure.
- **Constraints**: Reuse `fetchImageCached`/cache (FR-009). Per-asset failure non-blocking (FR-004).
