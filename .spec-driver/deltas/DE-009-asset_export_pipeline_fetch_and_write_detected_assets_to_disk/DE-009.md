## 1. Summary & Context
- **Technical Spec**: SPEC-001 ‚Äì Figma fetch and normalization system
- **Implementation Plan**: [IP-009](./IP-009.md)
- **Change Drivers**: IMPR-005 ‚Äî smoke testing revealed `assets/` directory is always empty

## 2. Motivation
- Asset detection (FR-011) correctly identifies bitmap fills and complex vectors during normalization, but the export pipeline stops there.
- `orchestrate.ts` hardcodes `assets: []`; no code collects, fetches, or writes detected assets.
- Users expect exportable assets (icons, images, complex vectors) to appear in the output directory.

## 3. Scope & Objectives
- **Primary Outcomes**:
  - Collect export-suggested nodes from normalized tree after expansion
  - Batch-fetch via Figma image export endpoint (FR-004 generalised to descendant nodes)
  - Write to `assets/` with stable, meaningful filenames (FR-015)
  - Populate `manifest.outputs.assets` with written paths
- **Operational Constraints**: Reuse existing `fetchImageCached` / cache infrastructure (FR-009). Per-asset failures must not block the pipeline (FR-004 error contract).

## 4. Out of Scope
- Token name resolution (IMPR-006 ‚Äî separate delta)
- CLI progress feedback (IMPR-004 ‚Äî separate delta)
- Asset format negotiation (bitmap‚ÜíPNG, vector‚ÜíSVG is sufficient for v1)

## 5. Approach Overview
- **System Touchpoints**: `orchestrate.ts`, `output/write.ts`, `output/manifest.ts`, `figma/fetch-image.ts`, `schemas/manifest.ts`
- **Key Changes**:
  - New `collectExportableAssets(node)` tree-walk utility
  - Generalise `fetchImageCached` to accept arbitrary node IDs (or add batch variant)
  - `writeAssets()` handler in `output/write.ts`
  - Wire collection ‚Üí fetch ‚Üí write into `orchestrate()`
  - Update manifest schema and builder

## 6. Verification Strategy
- **Requirements Coverage**: Unit tests for asset collection, integration test for write pipeline, smoke test on real document
- **Acceptance Criteria**:
  - `artifacts/assets/` contains exported files for nodes with `exportSuggested: true`
  - `manifest.outputs.assets` lists all written paths
  - Per-asset fetch failure is recorded in manifest errors, does not block pipeline
  - All existing tests pass (no regression)

## 7. Risks & Mitigations
- **Risk**: Figma image export rate limiting on batch requests ‚Äì *Likelihood*: medium ‚Äì *Impact*: medium ‚Äì *Mitigation*: Use existing `p-limit` concurrency control, respect cache
- **Risk**: Large documents produce many exportable assets ‚Äì *Likelihood*: low ‚Äì *Impact*: medium ‚Äì *Mitigation*: Respect existing `maxExpansionTargets` config; consider an asset cap if needed

## Implementation Notes ‚Äî Phase 02

### What's done
- **2.1**: `OutputArtifacts.assets` field added (`readonly FetchedAsset[] | undefined`). `writeAssets()` iterates assets, calls `assetFileName()`, writes to `assets/` subdir via `writeBinaryFile`.
- **2.2**: `orchestrate.ts` wired: `collectAndFetchAssets()` helper collects targets from normalized tree, builds `ImageFetcher` adapter over `fetchImageCached`, calls `fetchAssets`, computes manifest paths via `assetFileName`. `resolveAssetFormatOverride()` maps `'auto'` ‚Üí `null`. Asset errors merged into manifest errors. `OrchestrateResult.assets` now `readonly FetchedAsset[]` (required, not optional).
- **2.3**: 3 write tests ‚Äî correct filenames, empty array, undefined assets.
- **2.4**: 4 orchestrate tests (VT-038) ‚Äî manifest populated, result.assets present, maxAssets=0, no-exportable-nodes case. Used raw node with `fills: [{ type: 'IMAGE', imageRef: 'img:hero' }]` to trigger `exportSuggested` through normalization.
- **2.5**: `mise run check` ‚Äî 817/817 tests, zero lint, typecheck clean.

### Surprises / Adaptations
- `@typescript-eslint/return-await` required `return await` in `collectAndFetchAssets` ‚Äî the `return fetchAssets(...)` pattern triggered it because the function is `async`.
- Phase sheet said `writeAssets` returns `string[]` for manifest, but asset paths are computed in orchestrate (manifest is built before `writeOutput`). `writeAssets` is fire-and-forget void; paths computed independently via same `assetFileName` call. This is intentional duplication ‚Äî both sites use the same deterministic function.
