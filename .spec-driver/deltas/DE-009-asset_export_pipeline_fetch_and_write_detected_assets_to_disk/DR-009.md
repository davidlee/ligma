# DR-009 – Asset export pipeline — fetch and write detected assets to disk

Full design document: [docs/plans/2026-03-12-DE-009-asset-export-pipeline-design.md](../../../docs/plans/2026-03-12-DE-009-asset-export-pipeline-design.md)

## Summary

Complete the asset export pipeline by adding collection, fetching, and writing
of detected assets to disk. Asset detection (FR-011) already works; this delta
wires it through to file output (FR-015) using the existing image export
infrastructure (FR-004).

## Key Decisions

- **DEC-001**: Kind-driven format by default (bitmap→PNG, svg→SVG, mixed→both)
- **DEC-002**: `--asset-format` CLI flag overrides for all assets
- **DEC-003**: Filenames: `{sanitized-name}-{node-id}.{ext}` — readable + unique
- **DEC-004**: Reuse `fetchImageCached` — same concurrency as expansion
- **DEC-005**: Per-asset failure is non-blocking, recorded in manifest errors
- **DEC-006**: `--max-assets` flag, default 20, confidence-based priority
- **DEC-007**: New `src/assets/` module (collect.ts + fetch.ts)

## Code Impact

| Path | Change |
|------|--------|
| `src/assets/collect.ts` | **New** — tree walk, prioritise, cap |

| `src/orchestrate.ts` | Wire collect → fetch → manifest → write |
