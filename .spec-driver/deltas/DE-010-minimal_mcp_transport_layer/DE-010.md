## 1. Summary & Context
- **Product Spec**: [PROD-001](.spec-driver/product/PROD-001/PROD-001.md) — Figma fetch and normalization pipeline
- **Technical Spec**: [SPEC-001](.spec-driver/specs/tech/SPEC-001/) — Figma fetch and normalization system
- **Implementation Plan**: [IP-010](./IP-010.md)
- **Change Drivers**: Brief Phase 5 ("optional thin MCP wrapper"), PROD-001.NF-005 (library must be importable for MCP)

## 2. Motivation
- The core library (`orchestrate`) already produces the full artifact bundle. A thin MCP server exposes the same capability to Claude Desktop and other MCP-compatible agents without duplicating logic.
- The brief explicitly defers MCP to a later phase and mandates zero business logic in the MCP layer.

## 3. Scope & Objectives
- **Primary Outcomes**:
  - 6 granular MCP tools per brief Phase 5: `figma_get_node`, `figma_get_outline`, `figma_get_render`, `figma_list_assets`, `figma_get_asset`, `figma_get_assets`
  - MCP server entry point (`src/mcp.ts`) over stdio transport
  - Session extraction (`src/session.ts`) shared by CLI and MCP
  - 2 new CLI subcommands: `list-assets`, `get-asset`
- **Operational Constraints**: Zero business logic in MCP layer — transport only
- **Dependencies**: DE-001 through DE-006 (core pipeline, complete)

## 4. Out of Scope
- Streaming or incremental results
- MCP resources
- HTTP/SSE transport
- MCP-specific caching or state management
- Any business logic in the MCP layer

## 5. Approach Overview
- **System Touchpoints**: `src/session.ts` (new), `src/mcp.ts` (new), `src/orchestrate.ts` (refactor), `src/cli.ts` (subcommands), `package.json`
- **Key Changes**:
  - Extract `createSession()` from orchestrate for shared client/cache setup
  - Add `@modelcontextprotocol/sdk` dependency
  - Implement 6-tool MCP server delegating to pipeline primitives
  - Add `list-assets` and `get-asset` CLI subcommands
  - Add `ligma-mcp` bin entry
- **Rollout**: Ship alongside CLI; no migration needed

## 6. Verification Strategy
- **Requirements Coverage**: PROD-001.NF-005 — verified by MCP server calling pipeline primitives and returning results
- **Acceptance Criteria**:
  - MCP server starts and registers 6 tools via stdio transport
  - Each tool delegates to the correct pipeline primitives
  - Text/JSON tools return inline; image tools write to disk and return paths
  - CLI subcommands produce correct output
  - Zero business logic in MCP layer (audit-verifiable)
  - Server fails fast if `FIGMA_TOKEN` is unset

## 7. Risks & Mitigations
- **Risk**: MCP SDK churn — *Likelihood*: low — *Impact*: low — *Mitigation*: Pin SDK version, minimal API surface
- **Risk**: orchestrate refactor breaks existing tests — *Likelihood*: certain — *Impact*: low — *Mitigation*: Phase 1 refactors and fixes tests before MCP work
