## 1. Summary
- **Delta**: DE-010 — Minimal MCP transport layer
- **Design Revision**: [DR-010](./DR-010.md)
- **Specs Impacted**: PROD-001, SPEC-001
- **Desired Outcome**: 6-tool MCP server over stdio + 2 new CLI subcommands, sharing a common session layer with the existing pipeline.

## 2. Context & Constraints
- **Current Behaviour**: Pipeline accessible only via CLI. `orchestrate()` builds client/cache internally.
- **Target Behaviour**: Shared `createSession()` used by CLI, MCP, and future consumers. MCP server exposes 6 granular tools. CLI gains `list-assets` and `get-asset` subcommands.
- **Dependencies**: Core pipeline (DE-001 through DE-006, complete). DE-009 (assets, in-progress) — asset tools depend on `collectExportTargets` and `fetchAssets` which already exist.
- **Constraints**: Zero business logic in MCP layer. stdio only. `FIGMA_TOKEN` env var only.
