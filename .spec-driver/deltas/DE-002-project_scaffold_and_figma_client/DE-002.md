## 1. Summary & Context
- **Product Spec**: PROD-001 – Figma fetch and normalization pipeline
- **Technical Spec**: SPEC-001 – Figma fetch and normalization system
- **Implementation Plan**: [IP-002](./IP-002.md)
- **Change Drivers**: DE-001 planning delta

## 2. Motivation
Foundation delta. Establishes the TypeScript project structure, CLI entry point, Figma URL parser, API client with authentication, node subtree fetching, image export, artifact directory output, and manifest generation. Without this, no other delta can execute.

## 3. Scope & Objectives
- **Primary Outcomes**:
  - Working CLI that accepts a Figma URL and token (FR-001, NF-005)
  - URL parser extracting fileKey + nodeId (FR-001)
  - Authenticated Figma API client with retry (FR-002, FR-003, NF-004)
  - Node subtree fetch with depth/geometry options (FR-002)
  - PNG/SVG image export (FR-004)
  - Artifact directory structure and manifest output (FR-013)
  - Typed error hierarchy: FigmaUrlParseError, FigmaAuthError, FigmaNotFoundError, FigmaRateLimitError, FigmaRenderError, NormalizationError (FR-017)
  - Minimal dependency stack (NF-006)
- **Dependencies**: None (first implementation delta)

## 4. Out of Scope
- Normalization (DE-003, DE-004)
- Outline/context generation (DE-005)
- Selective expansion, caching (DE-006)

## 5. Approach Overview
- **System Touchpoints**: `src/orchestrate.ts`, `src/cli.ts`, `src/config.ts`, `src/errors.ts`, `src/figma/*`, `src/output/write.ts`, `src/output/manifest.ts`, `src/schemas/raw.ts`, `src/schemas/manifest.ts`, `src/util/*`
- **Key Changes**:
  - Project init (tsconfig, package.json, vitest config)
  - URL parser with Zod validation (SPEC-001.FR-001)
  - HTTP client with auth header injection (SPEC-001.FR-002)
  - Node fetch + image export endpoints (SPEC-001.FR-002, FR-003)
  - Manifest and directory writer (SPEC-001.FR-012)

## 6. Verification Strategy
- Unit tests for URL parsing (all URL formats, error cases)
- Unit tests for auth header generation
- Integration tests for client (mocked HTTP)
- Unit tests for manifest generation
- CLI smoke test

## 7. Risks & Mitigations
- **Risk**: Figma API response shapes differ from docs – *Likelihood*: low – *Impact*: medium – *Mitigation*: Zod schemas with `.passthrough()` for unknown fields

## 8. Follow-ups & Tracking
- **Future Deltas**: DE-003 (normalization), DE-004 (semantics), DE-005 (output), DE-006 (expansion)
