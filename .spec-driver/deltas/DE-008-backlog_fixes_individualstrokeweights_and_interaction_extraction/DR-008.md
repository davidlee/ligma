# DR-008 – Per-side Stroke Weights & Interaction Extraction

## 1. Executive Summary
- **Delta**: [DE-008](./DE-008.md)
- **Status**: approved
- **Last Updated**: 2026-03-11
- **Synopsis**: Fix per-side stroke weight extraction and add interaction normalization.

## 2. Problem & Constraints
- **Current Behaviour**: `extractAppearance()` drops `individualStrokeWeights`, producing incorrect uniform weight. `interactions` array dropped entirely.
- **Drivers**: ISSUE-001 (p2, data fidelity), ISSUE-002 (p3, missing extraction)
- **Constraints**: Additive schema changes only for interactions. Stroke weight is a contained breaking change (no downstream consumers).
- **Out of Scope**: Full interaction simulation, variable-driven bindings, media triggers, transition metadata.

## 3. Architecture Intent
- **Target Outcomes**:
  - Per-side stroke weights preserved in normalized output (FR-006)
  - Figma interactions extracted as compact normalized data
- **Guiding Principles**: Compact over exhaustive. Lossy normalization with explicit warnings for unsupported types.

See full design: [docs/plans/2026-03-11-DE-008-stroke-weights-and-interactions-design.md](../../docs/plans/2026-03-11-DE-008-stroke-weights-and-interactions-design.md)

## 6. Design Decisions & Trade-offs
- **DEC-030**: Stroke weight as discriminated union `{ uniform: true, value }` / `{ uniform: false, top, right, bottom, left }` — matches `CornerRadius` precedent.
- **DEC-031**: Action kind derived from `action.navigation` for node-targeting actions — matches Figma's actual REST model where navigation is a subtype.
- **DEC-032**: `interactions: NormalizedInteraction[] | null` — bare array, no wrapper. No metadata to justify a wrapper object.
- **DEC-033**: Lossy normalization boundary — `SET_VARIABLE`, `SET_VARIABLE_MODE`, `CONDITIONAL`, `UPDATE_MEDIA_RUNTIME`, `ON_MEDIA_HIT`, `ON_MEDIA_END` all normalize to `unknown` with warnings.
