## 1. Summary
- **Delta**: DE-006 — Selective expansion and caching
- **Specs Impacted**: SPEC-001, PROD-001
- **Design Revision**: DR-006
- **Desired Outcome**: Pipeline supports file-based caching of Figma API responses and single-pass selective expansion of depth-truncated or geometry-deficient nodes

## 2. Context & Constraints
- **Current Behaviour**: Single-pass fetch at configured depth, no caching, no expansion. Nodes at depth boundary silently lose children.
- **Target Behaviour**: FR-014 (file-based caching), FR-015 (selective expansion with configurable triggers)
- **Dependencies**: DE-003 (normalization engine, completed), DE-002 (Figma client, completed)
- **Constraints**: Normalize must remain pure (DEC-033). Single expansion pass only (DEC-036). Figma API depth >= 1 (DEC-038).

## 5. Phase Detail Snapshot
- **Design Revision**: `DE-006/DR-006.md`
- **Active Phase Sheet**: `phases/phase-01.md` (first to execute)
- **Parallelisable Work**: P01 and P02 are largely independent (expand/ doesn't depend on cache/ at the unit level), but P01 is sequenced first because cache-aware fetch helpers are needed by P03 integration. Triggers and merge (P02) can be developed concurrently if multiple agents are available.

## 6. Testing & Verification Plan

- **New Cases**: 16 VTs (VT-023 through VT-038)
- **Tooling/Fixtures**: Mock FigmaClient for cache tests, synthetic raw node trees for merge tests, mock filesystem for cache store tests
- **Regression**: VT-035 explicitly verifies expansion-disabled path matches pre-DE-006 behavior

## 7. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
| --- | --- | --- | --- |
| Expansion loop fetches too many children | medium | medium | maxTargets cap (default 10), single-pass only, p-limit concurrency |
| Cache staleness | low | low | Version-aware keys, explicit invalidation, no TTL |
| Depth-boundary false positives (childless containers) | low | negligible | Harmless refetch confirms no children; no downstream impact |
| Atomic write race on crash | low | negligible | Local dev cache; metadata is debugging aid, not correctness requirement |
