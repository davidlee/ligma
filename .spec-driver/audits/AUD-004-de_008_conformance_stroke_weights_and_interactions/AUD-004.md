## Findings

### F-001: Interactions extraction has no owning requirement in SPEC-001

- **Observation**: FR-006 covers appearance normalization (fills, strokes, effects, corner radius). Interaction extraction is a distinct concern — not appearance data. No existing FR covers it.
- **Outcome**: Implementation is correct and tested, but lacks a formal requirement in SPEC-001.
- **Disposition**: `spec_patch` — add a new FR to SPEC-001 for interaction extraction. This is a straightforward addition to the normalization engine capability, not an authority/scope change requiring a revision.

### F-002: SPEC-001 normalization-engine capability list missing interactions

- **Observation**: SPEC-001 `capabilities.normalization-engine.responsibilities` does not list interaction extraction.
- **Outcome**: Capability list is stale relative to implementation.
- **Disposition**: `spec_patch` — add interaction extraction to normalization-engine responsibilities, tied to the new FR.

## Recommendations

1. Patch SPEC-001: add FR for interaction extraction, update normalization-engine capability
2. All other findings aligned — no revision or follow-up work needed
3. Hand off to `/close-change` after spec patch
