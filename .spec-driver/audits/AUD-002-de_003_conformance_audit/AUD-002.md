## Scope

Conformance audit for DE-003 (Core normalization engine) against PROD-001 FR-005/006/008/009, NF-001/003 and SPEC-001 FR-004–009, NF-001–003.

## Observations

### FR-005 (normalized model) — CONFORMS

`normalize/node.ts` produces recursive `NormalizedNode` trees with all specified fields: id, name, type, role, visible, bounds, rotation, hierarchy, layout, appearance, text, component, variables, asset, semantics, children, diagnostics. Schema in `schemas/normalized.ts` matches DR-003 §4 contracts. DE-004 placeholder fields (component, variables, asset, role, semantics) are correctly null/default.

Evidence: VT-012 (26 tests), `schemas/normalized.ts` (308 lines, 30+ Zod schemas).

### FR-006 (type classification) — CONFORMS

`normalize/classify.ts` maps all documented Figma types to `NormalizedNodeType`. Image detection via fills, mask detection via `isMask`, unmapped types → `"unknown"`.

Evidence: VT-008 (30 tests).

### FR-008 (layout normalization) — CONFORMS

`normalize/layout.ts` normalizes to: mode (none/horizontal/vertical/grid/absolute), sizing (fixed/fill/hug/unknown per axis), alignment (main/cross), padding, gap, wrap, grid subobject, constraints, position, clipsContent. Diagnostic warnings for edge cases (fill outside auto-layout).

Evidence: VT-009 (36 tests).

### FR-009 (text normalization) — CONFORMS

`normalize/text.ts` extracts: content, charactersLength, style (fontFamily/weight/size/lineHeight/letterSpacing/textCase/alignment), color, tokenRefs, semanticKind, truncation. DimensionValue used for lineHeight/letterSpacing to preserve unit info.

Evidence: VT-011 (29 tests).

### NF-001 (representation efficiency) — CONFORMS (RE-002)

Original ">50% reduction" revised by RE-002 to two-part metric. Implementation satisfies both: (a) fixed 17-field top-level set < raw field count, (b) normalized size ≤2.0x raw on representative fixture (measured: 1.28x on real Figma data).

Evidence: VT-013 (2 tests).

### NF-003 (heuristics as pure functions) — CONFORMS

All extractors are pure functions: `(FigmaNode) → ExtractorResult<T>`. No shared mutable state. Each tested in isolation.

Evidence: VT-008/009/010/011 (125 tests across 4 suites).
