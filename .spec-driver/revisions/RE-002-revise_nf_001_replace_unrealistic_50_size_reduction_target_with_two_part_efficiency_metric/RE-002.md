## 1. Context

- **Why**: DE-003 Phase 3 implementation revealed that the original ">50% size reduction" target in NF-001 is empirically unreachable. Real Figma data (37-node frame, 10KB raw after Zod parse) produces normalized output that is ~28% *larger*, not 50% smaller. The normalized schema intentionally adds per-node structural metadata (hierarchy paths, diagnostics, semantic flags, DE-004 null placeholders) that serves the actual product goal — agent-useful structure — at a byte cost that exceeds what field stripping saves. The original target conflated "fewer bytes" with "more useful structure." 344/345 tests pass; VT-013 is the sole failure.
- **Source Specs**: PROD-001, SPEC-001
- **Destination Specs**: PROD-001, SPEC-001 (in-place patch, no movement)
- **Requirements Affected**: PROD-001.NF-001, SPEC-001.NF-001 (rewritten in place)

## 2. Related Artefacts

- **Deltas**: DE-003 (blocked by this revision — VT-013 cannot pass under old target)
- **Design Revisions**: DR-003 §2 constraint ("Output must be >50% smaller than raw input") requires update
- **Verification**: VT-013 requires rewrite to match new metric

## 3. Summary of Changes

- **PROD-001 NF-001**: Replace ">50% reduction in token count" with two-part efficiency metric: (a) schema-shape simplification — fixed, implementation-oriented top-level field set smaller than raw, (b) total size ceiling — JSON.stringify length must not exceed 2.0x raw input on representative fixtures. Semantic clarity and implementation utility take precedence over raw byte reduction.
- **SPEC-001 NF-001**: Mirror the PROD-001 change. Replace ">50% smaller than raw input in token count" with same two-part metric.
- **DR-003 §2**: Update the NF-001 constraint reference from ">50% smaller" to the new metric.
- **VT-013**: Rewrite test to assert (a) normalized top-level field count is bounded and (b) size ratio ≤ 2.0x on representative fixture.

## 4. Consequences

- **Positive**: Metric now measures what actually matters — local simplification and bounded overhead. Unblocks DE-003 Phase 3 completion. Prevents future regressions via fixture-based size tracking.
- **Positive**: Leaves room for a future percentile rule (e.g., "median ratio across representative fixtures ≤ 1.5x") without blocking current work.
- **Negative**: Loosens the original aspirational target. Acceptable because the original was set before schema design and was empirically wrong.
- **Testing impact**: VT-013 rewritten. All other VTs unaffected.
