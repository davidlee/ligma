## 1. Context
- **Why**: DR-004 (DE-004) design work identified that FR-007 needs concrete, testable heuristic sub-requirements to guide implementation. The existing FR-007 defines role vocabulary and conservative-preference posture but lacks the specific inference rules needed for deterministic, testable implementation.
- **Source Specs**: PROD-001
- **Destination Specs**: PROD-001 (in-place expansion)
- **Requirements Affected**: PROD-001.FR-007 (expanded with 8 heuristic sub-requirements)

## 2. Related Artefacts

## 3. Summary of Changes
- **Specs Updated**:
  - PROD-001: FR-007 expanded from role-vocabulary-only to include 8 concrete heuristic rules (container, stack, text semantic, button-like, input-like, icon, export-worthy asset, noise reduction)
- **Requirements**:
  - PROD-001.FR-007: no move/rename — in-place expansion. Each heuristic rule becomes a testable sub-requirement within FR-007. Confidence indicator and provenance trail added as mandatory inference outputs.
- **Responsibilities / Relations**: No change to capability ownership. The `normalization` capability in PROD-001 already covers FR-007. NF-003 (pure function heuristics) cross-referenced explicitly in the new text.

## 4. Consequences
- **Positive**: Each heuristic rule is now individually testable. Implementation has unambiguous acceptance criteria. Conservative posture preserved — `unknown` with diagnostic is the safe fallback.
- **Negative**: FR-007 is now substantially longer. Heuristic rules may need revision as real-world Figma patterns are encountered.
- **Testing impact**: Test suites for role inference must cover all 8 heuristic categories. Existing VT coverage for FR-007 (if any) needs expansion.
